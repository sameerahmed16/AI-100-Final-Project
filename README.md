# AI 100 Final Project

This project is based on my AI 100 midterm diabetes classification model.  
For the final project, I reused the original system and intentionally created 10 bug cases to study how changes in the code, model, preprocessing, training, and evaluation can affect the behavior of an AI system.

## Project Contents

- **Final Report.pdf**  
  Final written report for the project.

- **Cases.csv**  
  Spreadsheet containing the 10 bug cases, reflections, GenAI feedback, and revised reasoning.

- **Final Project/**  
  Main project folder containing the code and case files.

## Inside Final Project

- **base_code/**  
  Clean working version of the original diabetes classification project.

- **bug_cases/**  
  Folder containing 10 bug cases (`case_01` to `case_10`).

## Inside Each Case Folder

Each case folder includes:
- `buggy_train.py` – buggy version of the code
- `fixed_train.py` – corrected version of the code
- `train.py` – main file used for that case
- `diabetes.csv` – dataset used for the case
- `run_bug_output.txt` – output/error from the buggy version
- `run_fixed_output.txt` – output from the fixed version
- `genai_prompt_and_response.txt` – prompt and recorded GenAI response
- `evidence.md` – notes about the case
- `results/` – generated plots and model outputs
